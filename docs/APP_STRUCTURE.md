# QRIOUS 구조 설계서

## 핵심 원칙
- **사장님** → 앱 다운로드 (PWA 또는 추후 React Native)
- **손님** → QR 스캔 → 브라우저에서 바로 열림 (다운로드 없음)

---

## 전체 구조

```
QRIOUS
├── 손님용 웹 (브라우저, QR 스캔)
│   └── /{locale}/order/{restaurantId}/{tableId}
│       ├── 메뉴판 (배민 스타일 UI)
│       ├── 메뉴 상세 + AI 도슨트
│       ├── 장바구니 → 주문 → 결제
│       └── 주문 완료 + 비공개 리뷰
│
└── 사장님용 앱 (PWA → 추후 React Native)
    └── /{locale}/dashboard/{restaurantId}
        ├── 실시간 주문 접수
        ├── 메뉴 관리 (추가/수정/삭제/품절)
        ├── 테이블 & QR 관리
        ├── 레스토랑 설정 (로고, 이름, 테마색)
        └── 주간 AI 리포트
```

## 손님용 웹 — UI 방향 (배달의민족 스타일)

### 레이아웃 구조
```
┌─────────────────────────────┐
│  ← 가게명              🔍 🛒 │  ← 헤더 (가게명 + 검색 + 장바구니)
├─────────────────────────────┤
│ [전체] [메인] [사이드] [음료] │  ← 카테고리 탭 (가로 스크롤)
├─────────────────────────────┤
│ 🏆 인기 있는 메뉴             │
│ ─────────────────────────── │
│ [인기1위] [사장님추천]         │
│ 메뉴 이름                🖼️  │
│ 설명 텍스트              [+] │
│ 가격                         │
│ ─────────────────────────── │
│ 메뉴 이름                🖼️  │
│ ...                          │
└─────────────────────────────┘
│      장바구니 바 (하단 고정)   │
└─────────────────────────────┘
```

### 배민 스타일 핵심 요소
- 흰 배경 + 회색 구분선
- 메뉴 카드: 텍스트(왼) + 이미지(우, 80x80) + `+` 원형 버튼
- 인기 뱃지: `인기 1위`, `사장님 추천` (회색 pill)
- 카테고리 탭: 상단 고정, 가로 스크롤
- 섹션 헤더: `가장 인기 있는 메뉴` + 설명 문구

## 사장님용 앱 — PWA 설계

### 현재 상태
- PWA manifest 이미 적용 (`manifest.json`, theme_color #ec751c)
- A2HS (홈 화면 추가) 가능

### 앱처럼 느끼게 하는 추가 작업
1. **설치 유도 배너** — `/dashboard` 접속 시 "홈 화면에 추가하세요" 안내
2. **스플래시 스크린** — 앱 로딩 화면
3. **네이티브 앱 느낌 UI** — 바텀 네비게이션 탭
   ```
   [주문] [메뉴] [테이블] [설정]
   ```
4. **오프라인 지원** — 기본 화면은 오프라인에서도 표시

### 추후 React Native 전환 시
- Expo + Supabase JS SDK 그대로 재사용 가능
- API 엔드포인트 변경 없음
- 앱스토어/플레이스토어 배포

## 언어 전략

### 손님용 웹
- 다국어 유지 (한/영/중/일)
- QR URL에 locale 포함: `/{locale}/order/...`
- 언어 선택기 상단 노출

### 사장님용 앱
- 한국어 단일 언어로 단순화
- 추후 글로벌 확장 시 다국어 추가
